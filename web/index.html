The viewer reads ./stats.db from the directory it was started in, but the database (or its tables) doesn't exist yet.

Create the tables, then run the scanner from the repo root:

$ sqlite3 stats.db
sqlite> CREATE TABLE seen_mails (mail_id string);
sqlite> CREATE TABLE senders (sender string, mails_sent int);
sqlite> .quit
$ cargo run
